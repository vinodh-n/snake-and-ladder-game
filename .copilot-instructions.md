# GitHub Copilot Instructions for Snake and Ladder Game

## Project Overview
This is an **Interactive Snake and Ladder Board Game** built with vanilla HTML, CSS, and JavaScript. It's a web-based implementation of the classic board game where two players race to reach position 100, encountering snakes and ladders along the way.

## Key Features
- **Two-Player Game**: Turn-based gameplay for two players
- **Dice Rolling**: Random 1-6 dice roll animation
- **Snakes & Ladders**: 8 ladders and 7 snakes on the board
- **Move History**: Tracks last 10 moves with descriptions
- **Game Rules Modal**: Built-in rules and how-to guide
- **Responsive Design**: Works on desktop, tablet, and mobile
- **Winner Celebration**: Modal announcement when a player wins
- **Real-time Updates**: Live position tracking for both players
- **Python Implementation**: Full CLI version with testing

## Game Configuration

### Board Setup
- **Board Size**: 100 squares (10x10 grid)
- **Grid Pattern**: Zigzag numbered pattern (typical snake and ladder style)
- **Starting Position**: Both players start at position 1
- **Winning Condition**: First to reach exactly position 100

## Game Logic Overview

### Core Functions

#### `initializeGame()`
- Creates the board
- Sets up event listeners
- Updates the UI

#### `rollDice()`
- Generates random number (1-6)
- Animates the dice
- Triggers player movement

#### `movePlayer()`
- Calculates new position
- Checks for snake/ladder encounters
- Validates move (can't go beyond 100)
- Checks win condition
- Adds move to history

#### `switchPlayer()`
- Alternates between Player 1 and Player 2
- Resets UI for next turn

#### `updateBoard()`
- Positions player markers on board
- Updates position displays
- Removes old positions

#### `addMoveToHistory(message)`
- Stores move details
- Maintains last 10 moves
- Updates moves list display

### Game State Structure
```javascript
gameState = {
    players: [
        { id: 1, position: 1, element: 'p1-position' },
        { id: 2, position: 1, element: 'p2-position' }
    ],
    currentPlayer: 0,        // 0 = Player 1, 1 = Player 2
    diceValue: 0,
    gameOver: false,
    moveHistory: [],
    diceRolled: false        // Prevents multiple rolls per turn
}
```

## Common Issues & Solutions

### Issue: Game not responsive on mobile
**Solution**: Check CSS media queries in style.css. Ensure viewport meta tag is in HTML.

### Issue: Dice animation loops infinitely
**Solution**: Verify timeout in rollDice() is set to 600ms and animation class is removed.

### Issue: Player appears on wrong square
**Solution**: Check zigzag board creation logic in createBoard(). Rows with even indices go left-to-right.

### Issue: Board doesn't look like traditional snake and ladder board
**Solution**: The board now uses a proper 10x10 grid with numbered squares following the traditional zigzag pattern. Numbers increase in a specific pattern for authentic gameplay.

## Board Layout Details

### Traditional Snake and Ladder Board Pattern
The board follows the classic 10x10 snake and ladder pattern:
- **Row 10** (Bottom): Squares 1-10 (left to right)
- **Row 9**: Squares 20-11 (right to left)
- **Row 8**: Squares 21-30 (left to right)
- **Row 7**: Squares 40-31 (right to left)
- And so on, alternating direction for each row

### Visual Indicators
- **Squares**: White background with position number
- **Ladders**: Green background with 🪜 icon
- **Snakes**: Red background with 🐍 icon
- **Player 1**: Blue marker (🔵) on the square
- **Player 2**: Yellow marker (🟡) on the square

## Python Implementation Details

### Classes

#### `Player`
Represents a single player in the game.

**Attributes:**
- `id`: Player identifier (1 or 2)
- `name`: Player's name
- `position`: Current position on board (1-100)
- `moves_count`: Total number of moves made
- `ladder_climbs`: Number of ladders climbed
- `snake_slides`: Number of snakes encountered

#### `Board`
Represents the game board with snakes and ladders.

**Constants:**
- `BOARD_SIZE`: 100 squares
- `GRID_SIZE`: 10x10 grid
- `SNAKES`: Dictionary of snake positions
- `LADDERS`: Dictionary of ladder positions

#### `Game`
Main game controller managing gameplay.

**Attributes:**
- `board`: Board instance
- `player1`: Player 1 instance
- `player2`: Player 2 instance
- `current_player_id`: Current active player
- `status`: Game status (running/finished)
- `moves_history`: List of all moves
- `dice_rolls`: List of all dice rolls
